# Project Documentation

## 1. Project Overview
- Project name: hellokitty. [Verified]
- What it does: A browser-based Pomodoro-style study timer with a landing view, timer view, session progression (focus and break modes), mascot-driven UI states, and lightweight audio/visual feedback. [Verified]
- Intended purpose: Help users run timed focus sessions with clear mode transitions and a playful, calming interface. [Verified]
- Target user: Students or anyone using simple timed focus sessions. [Inferred]

- Start, Pause, Reset controls with dynamic button enabled/disabled mapping. [Verified]

- Session tracker with four strawberry markers. [Verified]
- Two-page flow (landing and timer) in one HTML document. [Verified]

## 5. How It Works
User flow:
1. App starts on landing page (navigateTo("landing") in main.js). [Verified]
2. User clicks "Let's Study!" to switch to timer page. [Verified]
3. Start begins interval countdown; Pause stops interval; Reset returns to study mode. [Verified]
4. On timer completion:
   - If in focus mode: increment sessionCount, update strawberries, trigger confetti, switch to short/long break.
   - If in break mode: return to focus mode (long break also resets sessionCount). [Verified]

State/data flow:
- Runtime state is stored in a single state object (timer handle, timeLeft, mode, isActive, sessionCount). [Verified]
- UI is updated from state through updateDisplay(), updateButtonsState(), updateStrawberryTracker(), updateMascotState(). [Verified]
- localStorage key kittyTimerState is written from saveState(), but current load behavior intentionally resets sessionCount to 0 on page load. [Verified]

Styling flow:
- Body class mode-study / mode-short-break / mode-long-break controls global background theme. [Verified]
- Mascot visual state class is toggled by JS; CSS defines base continuous motion and state modifiers. [Verified]

- src/main.js

## 9. Architecture Notes
- Architectural style: Single-document UI with two logical pages toggled via CSS classes; no router, no framework state manager. [Verified]
- Separation of concerns:
  - main.js handles app shell navigation and entry imports.
  - script.js handles timer domain logic and UI state sync.
  - CSS split between global/timer style file and landing-specific file. [Verified]
- Pattern used:
  - Central mutable state object plus explicit render/update functions. [Verified]
